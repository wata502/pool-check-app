- **修正リスク:** 低（クリーンアップの追加のみ）
- **優先度:** 中

### 指摘 HTML-1: XSS脆弱性 ― special_notes の未サニタイズ innerHTML 挿入

- **該当箇所:** 行1496-1497, 2035-2036
- **カテゴリ:** セキュリティ
- **問題の内容:** 確認画面（`showConfirm()`、行1497）と詳細表示（`showDetail()`、行2036）で `special_notes.replace(/\n/g,'<br>')` の結果を `innerHTML` に直接挿入している。ユーザーが特記事項に `<script>` タグや `<img onerror=...>` を入力した場合、スクリプトが実行される。Firebase経由で他端末にも伝播するため、格納型XSSになりうる。
- **推奨する対応:** テキストをHTML挿入する前にエスケープ関数を通す。例:
  ```javascript
  function escapeHtml(s) {
    const d = document.createElement('div');
    d.textContent = s;
    return d.innerHTML;
  }
  // 使用例
  escapeHtml(data.special_notes).replace(/\n/g,'<br>')
  ```
- **修正リスク:** 低（表示処理の変更のみ）
- **優先度:** 高

  ```javascript
    );
  })
  ```
- **修正リスク:** 低

### 全体の所見

本アプリケーションは、プール付属電気設備の点検業務をスマートフォンから入力し、Firebase経由でPC側のExcelファイルに自動書き込みするという複雑なワークフローを実現しており、全体として実用的なレベルで完成度が高い。

特に評価できる点として、pool_writer.py の冪等性チェック、SSE＋ポーリングの二重監視によるデータ欠落防止、COM操作のリトライ機構、weather_judge.js の純粋関数設計（外部ステートなし）、pool_mapping.json によるExcelセル位置の設定データ分離、Service Worker のキャッシュ戦略（ネットワーク優先＋キャッシュフォールバック）などが挙げられる。

最優先で対応すべきは、pool_settings.json の認証情報漏洩リスク（S-1）と、index.html の XSS 脆弱性（HTML-1, HTML-2）の2点。特にXSSはFirebase経由で他端末にも伝播する格納型XSSとなりうるため、HTMLエスケープ関数の導入を強く推奨する。SSEイベント処理の重複ロジック（HTML-5）もトースト二重表示やデータ不整合の原因となるため、早期の修正が望ましい。

その他の指摘はいずれも修正リスクが低く、段階的に対応しても既存動作を壊す可能性は小さい。
